# README Focus — app (Next.js Dashboard)

When updating the app README, focus on:

- **Getting started** — local dev setup, environment variables, running the dev server
- **Key pages and routes** — what the dashboard shows, main user flows
- **Tech stack** — Next.js version, key dependencies, styling approach
- **Project structure** — important directories under `src/`
- **Available scripts** — npm commands for dev, build, lint, test

Ignore purely internal refactors (moving files, renaming variables) unless they change the public API or developer workflow.

Keep the README concise and practical. Developers should be able to clone the repo and get running within minutes.
